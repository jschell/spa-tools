KQL Playground lets you query local CSV files using Kusto Query Language (KQL) — the same syntax used in Azure Data Explorer and Microsoft Sentinel — without uploading any data. Drop a CSV file onto the page or use the file picker, and the tool loads it directly into an in-browser DuckDB engine where it never leaves your machine.

Write KQL in the editor pane and press Shift+Enter (or the Run button) to execute. The tool supports the core Phase 1 operators: `where` for filtering rows, `project` to select columns, `take` / `limit` for row limits, `extend` to compute new columns, `summarize` to aggregate with optional `by` grouping, `count`, `distinct`, `order by`, `sort by`, and `top N by Column`. Column names appear in the sidebar and can be clicked to insert them into the editor, and the editor offers basic KQL keyword and column-name auto-complete.

Results are displayed in a scrollable table (up to 10,000 rows on screen) and can be exported to CSV. The PS Export button converts the active query into a PowerShell pipeline using `Import-Csv`, `Where-Object`, `Select-Object`, and `Sort-Object` equivalents for easy use in Windows automation scripts. The current query is persisted in the URL so you can bookmark or share a pre-filled playground link.
